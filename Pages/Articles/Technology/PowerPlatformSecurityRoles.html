Power Platform Security Roles

May 5th 2024 10 Min read

Types of Predefined Power Platform Roles

These roles represent common user tasks:

Environment Roles without Dataverse:

  • Environment Admin
  • Environment Maker

Environment Roles with Dataverse:

These roles cannot be edited but can be copied and modified:

  • System Admin (Instead of Environment Admin, user must have System Admin role for full admin privileges)
  • Environment Maker + System Customizer (to create apps and make changes to entities)
  • App Opener
  • Basic User
  • Delegate
  • Dynamics 365 Administrator
  • Environment Maker
  • Global Administrator
  • Global Reader
  • Office Collaborator
  • Power Platform Administrator
  • Service Deleted
  • Service Reader
  • Service Writer
  • Support User
  • System Administrator
  • System Customizer
  • Website App Owner
  • Website Owner

Power Platform admin center is used to configure access to Power Platform for Environments:

User Criteria:

  • Individual who has an active license
  • Enabled in Microsoft Entra ID
  • Member of the environment's security group

More information: Create users - Power Platform | Microsoft Learn

Definitions:

Types of Users:

  • Regular User: Synchronized with Microsoft Entra ID.
  • Application User: Built-in user account for integration and system back-end service support. Created during system provisioning.
  • System User: Built-in user account allowing customers to perform system updates via plug-ins.
  • Non-interactive User: Access mode for programmatic access between applications, no license required.
  • Support User: Placeholder for internal Microsoft support users, created by Dataverse.
  • Delegated Administrator: Placeholder user record for partners, generated by Dataverse.

More information on users: System and Application Users

Server-to-Server (S2S) apps:

Authenticate and communicate with Microsoft Dataverse using web applications and services.

More information: Server-to-Server Authentication

Team (optional):

Share business objects and collaborate with other business units.

More information: Manage Teams

Types of Teams:

  • Owner team: Owns records, has assigned security roles, grants full access rights, manually adds team members.
  • Access team: Doesn't own records, lacks assigned security roles, shares records with the team for Read, Write, and Append access.
  • Microsoft Entra group team: Similar to owner teams, can own records, dynamically adds/removes members based on Entra group membership.

Business Unit:

A logical grouping of related business activities mapped to an organization's department or division. Allows access to records within the business unit.

More information: Create and Edit Business Units

Parent: Child Business Unit:

Grants access to records in the business unit and its subordinates, typically for managers.

Organization:

Provides access to all records in the organization, usually reserved for managers.

Notes:

Use Entra Security Groups and Users, Teams, and Business Units to achieve Separation and Segmentation of data.

Source: Security Roles and Privileges

How to...

  • Assign licenses
  • Enable/Disable users to an environment
  • Assign Security Roles
  • Create or edit a Security Role
  • Create or configure a custom Security Role
  • Remove Power Platform Licenses from users

Best practices and strategies:

  • License assignment strategies
  • Reassign resources to a licensed user and remove the user from the Entra ID group if a licensed user leaves the organization.
  • Understand License Types: Familiarize yourself with the various license types offered by Power Platform.
  • Assess User Needs: Determine specific requirements of each user or group within your organization to assign appropriate licenses efficiently.
  • Optimize Licensing: Regularly review and adjust licenses based on user activity, ensuring necessary features without unnecessary costs.
  • Utilize Trial Periods: Use trial periods for new users or projects to assess needs before committing to long-term licenses.
  • Implement Role-Based Licensing: Assign licenses based on users' roles and responsibilities to ensure they have access to needed tools.
  • Monitor Usage: Track license usage through the Power Platform Admin Center to identify underutilized licenses or potential license violations.
  • Educate Users: Provide training and resources to maximize value of licenses and encourage adoption of Power Platform tools.
  • Consider External Users: Evaluate the need for external users to access Power Platform applications and explore options like guest access or dedicated licenses.
  • Stay Updated: Stay informed about changes in licensing policies and features to make informed decisions and optimize license management strategies.
